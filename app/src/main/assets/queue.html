Queue

Queue is an abstract data structure, somewhat similar to Stacks. Unlike stacks, a queue is open at both its ends. One end is always used to insert data (enqueue) and the other is used to remove data (dequeue). Queue follows First-In-First-Out methodology, i.e., the data item stored first will be accessed first. A real-world example of queue can be a single-lane one-way road, where the vehicle enters first, exits first. More real-world examples can be seen as queues at the ticket windows and bus-stops.

Limitation of linear queue/simple queue:

Suppose, we have inserted 5 elements now rear=4 and front=5,and we deleted 5 elements.
if we try to add more elements , the elements can't be inserted ,because in queue ,the new elements are always inserted from the rear end and read end indicated the last location of the queue .
Hence, though the space is there in the queue,we are unable to insert the elements.
To remove this problem,the 1st and obvious solution is that whenever element is deleted , shift all afterward elements to left by one position,but this is time consuming

Circular Queue

Circular queue is one in which insertion of new elements is done at the very first location of queue , if last location of queue is full
In other words, if we have queue of 5 elements then after inserting an element last(n-1=4th) location of queue next element will be inserted at very 1st location of queue.
If that location is empty Or It is queue in which 1st element just comes after the last element.
